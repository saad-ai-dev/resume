Changes Requested PR #2931 — feat: add 15 bounded-context PostgreSQL views
[P1] Billing view can attach the wrong Stripe subscription
File: sql/users_v_company_billing.sql#L84

The lateral join selects the newest row from StripeSubscriptions for the company instead of the subscription referenced by the chosen CompanyPaymentPlans row. If a company has historical or parallel subscriptions, the view can combine one plan with an unrelated Stripe record.

[P1] Listings views cannot expose real listing details because source data lives in MongoDB
Files: sql/listings_v_status.sql, sql/listings_v_search_keyword.sql

The PR adds PostgreSQL views for the listings domain, but the actual rich listing content is stored in MongoDB. These views can only expose sync/status metadata — consumers will assume coverage that does not exist.

[P2] Google insights materialized view drops soft deletes
File: sql/reporting_v_google_insights.sql#L75

Filtered with deletedAt IS NULL — soft-deleted rows simply disappear after refresh. CDC consumers receive no tombstone and can only recover through full rebuild.
Changes Requested PR #2673 — feat(datashake): switch daily cron from diff/jobId to from_date
[Critical] this binding bug in backupCronJobOnReviewSource

Arrow function calls this.postNewProfileCornJob(...). Arrow functions don't bind this — will throw at runtime. Tests pass only because the mock is on the controller object.

[Critical] Cron schedule mismatch creates data gaps

Daily skip uses (date-1) % 3 === 0 → fires on days 1, 4, 7, 10...
Backup cron uses */3 → fires on days 3, 6, 9, 12...
These sets are completely disjoint. On skip days, neither cron runs — recurring data gaps.

[Security] Unauthenticated route triggers paid API

/api/v1/datashake/cronJob has no auth middleware. Anyone who discovers the URL can trigger Datashake API calls, consuming paid quota.
